What's Wrong With Traditional Logs?

My logs contained tons of useful data that I can read but a computer couldn't easily understand.

So How Is Serilog Going to Help?

It's going to turn this log message

09:14:22 [Information] Processed { Latitude: 25, Longitude: 134 } in 034 ms.

into this JSON object

{
  "Position": {"Latitude": 25, "Longitude": 134}, 
  "Elapsed": 34
}

What Does the Code Look Like?

var position = new { Latitude = 25, Longitude = 134 };
var elapsedMs = 34;

log.Information("Processed {@Position} in {Elapsed:000} ms.", position, elapsedMs);

  • Log Level: Information
  • Message Template: "Processed {@Position} in {Elapsed:000} ms."
  • Property Values: position, elapsedMs
